# Figure 5: Mechanistic analysis of the learned policy

This directory contains the scripts and data required to reproduce 
**Figure 5** of the paper *Containment of Escaping Targets in 
Complex Networks*, along with the supplementary figure that 
extends the same mechanistic analysis to spatially localized 
targets (**Supplementary Fig. 7**).

Figure 5 reports the dynamics of an isolation episode along four 
metrics, each averaged over realizations and plotted as a function 
of normalized episode time:

- **isolated_ratio** — cumulative fraction of targets isolated from 
  the LCC
- **cost_benefit** — per-step LCC reduction caused by each removal
- **distances** — distance from each removed node to the nearest 
  target
- **local_betweenness** — target-relative local betweenness of each 
  removed node

The eight panels of the figure correspond to a $4$ metrics 
$\times$ $2$ topologies (Watts–Strogatz and Barabási–Albert) 
layout. Both target distributions (random and localized) are 
generated in a single CSV; which one is rendered is selected via 
a variable at the top of `draw_fig5_new.py`.


## Files

### Scripts

| Script | Role |
|--------|------|
| `comparison_method_realtime_track.py` | Runs the full mechanistic experiment. For each $(\textrm{topology}, \textrm{target distribution})$ regime, it tracks the four metrics across normalized episode time and averages them over $100$ independent simulations on $N = 1{,}024$ networks at $\lambda = 20$. Imports utility functions from `../fig3/comparison_method.py`. |
| `draw_fig5_new.py` | Renders the final Figure 5 (or its supplementary counterpart, depending on `TARGET_DIST`), combining the four metrics with side-by-side WS and BA topology blocks. Reads the CSV from `data/`. |

### Data (`data/`)

| File | Description | Source |
|------|-------------|--------|
| `realtime_tracking_n1024_lambda20.csv` | Time-resolved metric values (mean and SEM) for each method, topology, and target distribution. | Generated by `comparison_method_realtime_track.py` |

The CSV contains all four regimes (`BA-random`, `BA-localized`, 
`WS-random`, `WS-localized`) in a single tidy table; the rendering 
script selects rows by `TARGET_DIST`.


## Mapping to figures in the paper

A single rendering script produces both figures; which figure is 
rendered on a given invocation is controlled by the variable 
`TARGET_DIST` at the top of `draw_fig5_new.py`.

| `TARGET_DIST` | Renders as |
|---------------|------------|
| `'random'` | **Main-text Figure 5** |
| `'localized'` | **Supplementary Figure 7** |

preserved.


## Reproducing the figures

The repository ships with the pre-computed intermediate CSV in 
`data/`, so both figures can be reproduced directly from the 
shipped data.

To render **main-text Figure 5** (`random` targets):

1. Edit the top of `draw_fig5_new.py` to set `TARGET_DIST = 'random'`.
2. Run:

```bash
python draw_fig5_new.py
```

To render **Supplementary Figure 7** (`localized` targets):

1. Edit the top of `draw_fig5_new.py` to set `TARGET_DIST = 'localized'`.
2. Run the same command above.

Each invocation produces `draw_fig5_new_{TARGET_DIST}_new.pdf` and 
`.png` in this directory.

### Regenerating the data from scratch

To regenerate the time-resolved metric data from scratch, run:

```bash
python comparison_method_realtime_track.py
```

This produces `data/realtime_tracking_n1024_lambda20.csv`. Once 
the CSV is in place:

```bash
python draw_fig5_new.py
```


## Configuration

Key parameters can be adjusted at the top of each script:

- `comparison_method_realtime_track.py`:
  - `N_NODES = 1024` — synthetic network size
  - `TARGET_RATIO = 0.05`
  - `LAMBDA_EFF = 20` — fixed evasion factor
  - `SIMULATION_TIMES = 100` — independent simulations per condition
  - `NUM_BINS = 50` — number of normalized-time bins for averaging
  - `N_WORKERS`, `N_WORKERS_RL` — multiprocessing pool sizes
- `draw_fig5_new.py`:
  - `TARGET_DIST` — `'random'` (main Fig. 5) or `'localized'` 
    (Suppl. Fig. 7)
  - `T_FINAL_LO`, `T_FINAL_HI` — the time window over which the 
    inset bar charts compute their final-bin means
